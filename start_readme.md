デジタルクローン（Sato Clone）をご自身の環境で起動するには、用途に合わせていくつかのアプローチが用意されています。プロジェクト内のファイルを確認したところ、以下の3つの起動方法があります。

1. Web UI（ブラウザ）で起動する場合 [おすすめ]
一番直感的にチャット画面などでやり取りができる方法です。専用の起動スクリプトが用意されています。

ターミナルで対象のディレクトリ（/home/toshiaki/sato-clone-org）に移動し、以下のコマンドを実行します。

bash
./start_ui.sh
※自動的に必要なパッケージがインストールされ、Streamlitという機能を使ってブラウザでUIが開きます（またはURLが表示されるのでアクセスします）。

2. コマンドライン（CLI）で対話する場合
ブラウザを開かず、ターミナル上で直接文字のやり取りをしたい場合は、ローカルのCLIスクリプトを実行します。

bash
# 仮想環境を有効化（すでに有効になっている場合は不要）
source .venv/bin/activate
# CLIスクリプトを実行
python3 infrastructure/local_cli.py
※Windows側からWSLを経由して起動するための Sato_Clone_CLI.bat というファイルも用意されています。

3. バックエンドの本格稼働（Docker Compose）
複数のエージェント（PicoClaw Core）やMQTTブローカー、Telegramなどを連携させる本格的なシステム全体のインフラを立ち上げる場合です。

bash
docker compose up -d
まずは一番手軽な 1. Web UI（./start_ui.sh） や、ターミナルで実行できる 2. コマンドライン のどちらかを試していただくのが良いかと思います。
